Onboarding Práctica - Semana 1

Objetivo

Este repositorio tiene como objetivo servir como entorno de práctica para comprender la estructura de un monorepo, el uso de Git, la organización de aplicaciones y librerías, así como los fundamentos necesarios para trabajar con arquitecturas modernas basadas en Module Federation.

Estructura del repositorio

text
.
├── apps/
│   ├── shell/
│   ├── workspace/
│   ├── portal/
│   ├── infinity/
│   └── security/
├── libs/
│   ├── core/
│   ├── mf/
│   ├── shared/
│   ├── data-access/
│   ├── services/
│   ├── state/
│   ├── ui/
│   ├── util/
│   ├── i18n/
│   ├── pwa/
│   └── testing/
├── docs/
│   └── semana-1/
├── scripts/
└── README.md
```

 Descripción de carpetas

* apps/: Aplicaciones principales del proyecto.
* libs/: Librerías compartidas y reutilizables.
* docs/: Documentación del proyecto.
* scripts/: Scripts de automatización y utilidades.
* README.md: Información general del repositorio.
